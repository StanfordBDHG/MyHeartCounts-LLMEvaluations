# Nudge Evaluation Module

This module contains tools for evaluating and analyzing nudges generated by the `nudge-generation` module. These tools perform various linguistic and statistical analyses on generated nudge permutations.

For project overview and quick start, see the [root README](../README.md).

## Available Tools

### LLM-As-Judge (`llm-as-judge/`)

Evaluates a single nudge against the same question set and scoring rubrics used by the custom survey service, using OpenAI or SecureGPT backends reused from `nudge-generation`. Produces JSON and CSV artifacts for downstream analysis.

### Linguistic Analysis (`analyzeNudgeLinguistics.py`)

Performs automated linguistic analyses on generated nudges from CSV files. It extracts individual nudges (titles and bodies) from the JSON responses and computes comprehensive linguistic features.

## Setup

### 1. Install Python Dependencies

```bash
cd nudge-evaluation
pip install -r requirements.txt
```

### 2. Install spaCy English Model

The linguistic analysis tool requires the spaCy English language model:

```bash
python -m spacy download en_core_web_sm
```

## Usage

### Linguistic Analysis

The `analyzeNudgeLinguistics.py` script analyzes CSV files produced by `generateNudgePermutations.ts` in the `nudge-generation` module.

#### Basic Usage

```bash
# Analyze a file from the default location (data/generated/)
python analyzeNudgeLinguistics.py ../data/generated/nudge_permutations_results_gpt-5_sample_2.csv

# Analyze with custom output location
python analyzeNudgeLinguistics.py input.csv --output custom_output.csv

# Generate summary statistics
python analyzeNudgeLinguistics.py input.csv --summary

# Both output file and summary
python analyzeNudgeLinguistics.py input.csv --output analyzed_results.csv --summary
```

#### Command Line Arguments

- `input_file` (required) - Path to input CSV file from `generateNudgePermutations.ts`
- `--output`, `-o` - Output file path (default: `data/evaluated/{input}_with_linguistics.csv`)
- `--summary`, `-s` - Generate summary statistics CSV grouped by modelId

## Features Computed

For each nudge, the linguistic analysis script computes the following features:

### Body Features (7 features):
- `body_word_count` - Number of words in the body text
- `body_sentiment` - VADER sentiment compound score (-1 to 1)
- `body_action_verb_frequency` - Count of action verbs (excluding aux/copula)
- `body_temporal_reference` - Binary (0/1) indicating presence of temporal keywords or DATE/TIME entities
- `body_lexical_diversity` - Type-token ratio (unique tokens / total tokens)
- `body_exclamation_usage` - Count of exclamation marks
- `body_readability` - Flesch Reading Ease score

### Title Features (6 features):
- `title_word_count` - Number of words in the title text
- `title_sentiment` - VADER sentiment compound score (-1 to 1)
- `title_action_verb_frequency` - Count of action verbs (excluding aux/copula)
- `title_temporal_reference` - Binary (0/1) indicating presence of temporal keywords or DATE/TIME entities
- `title_exclamation_usage` - Count of exclamation marks
- `title_readability` - Flesch Reading Ease score

**Note:** Lexical diversity (TTR) is only computed for body text, not titles.

## Output Format

The script creates an expanded CSV where each row represents one nudge (instead of one permutation). The output includes:

- All original columns from the input CSV
- `nudge_index` - Index of the nudge within the response (0-6)
- `title` - The nudge title text
- `body` - The nudge body text
- All linguistic feature columns (as listed above)

By default, the output file is saved to `data/evaluated/{input_filename}_with_linguistics.csv`.

## Summary Statistics

When using the `--summary` flag, the script generates an additional CSV file with aggregated statistics grouped by `modelId`. For each model, it computes:

- Mean, standard deviation, minimum, and maximum for each linguistic feature
- Saved to `data/evaluated/summary_linguistics_by_model.csv` (or custom location if `--output` is specified)

## Technical Details

### Temporal Keywords

Temporal references are detected using:
`{"today", "tomorrow", "tonight", "morning", "evening", "daily", "each day", "every day", "this week", "next week", "in 10 minutes", "routine"}`

Plus any DATE or TIME entities identified by spaCy's named entity recognition.

## Troubleshooting

### spaCy Model Not Found

If you see an error about the spaCy English model:
```bash
python -m spacy download en_core_web_sm
```

### Input File Not Found

Make sure the CSV file exists. Generated CSV files are typically in `../data/generated/` relative to the `nudge-evaluation/` directory.

### Missing Dependencies

If you encounter import errors, ensure all dependencies are installed:
```bash
pip install -r requirements.txt
```
